# Anonymous Repository Checklist

## Files to include

- `run.py`
- `vgs=4.csv`
- `T=150.csv`
- `weights_exp156/`
- `README.md`
- `requirements.txt`
- `.gitignore`

## Files not to include

- training scripts
- internal notes
- raw data not intended for release
- author names or affiliations
- slides, drafts, review responses, or submission PDFs
- commit history with identifying metadata if avoidable

## Before making the repository public

- remove personal profile links from the repository description
- remove funding and acknowledgement text
- remove institution names from file paths and comments
- verify that figure titles, filenames, and comments do not contain author identifiers
- verify that the Git commit author name and email are anonymized

## Suggested release flow

1. Create a fresh anonymous GitHub account or use a clean repository with anonymized commit identity.
2. Upload the minimal files only.
3. Confirm that the repository page does not expose your identity.
4. If the venue allows it, generate an anonymous mirror with anonymous.4open.science.
5. Put only the anonymous link in the submission.
